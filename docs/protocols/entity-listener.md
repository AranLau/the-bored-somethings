# Entity Listener Protocol

> COGITATE 的会话级自动挂载机制：当 core-memory 中的关键实体在对话中被提及时，静默启动上下文标记。
> 此协议不抢 ELEMENT 的交互控制权。它在后台运行。

---

## 阶段 0：Session 启动注册

每个对话 session 开始时，执行：

1. 读取 `knowledge/core-memory.md` → "关键角色侧写"节
2. 提取所有人名/代号，注册为**监听目标**
3. 当前注册清单：冥后、爱人、火神、磐神、信使
4. 每个监听目标维护一个 `{提及次数, 最后提及轮次, R级别}` 的内部状态

此阶段**不输出任何内容**。用户感知不到监听器的存在。

## 阶段 1：首次提及 → R1 静默标记

当用户消息中首次出现任一监听目标的名字时：

- 从 core-memory 中调取该实体的上下文摘要
- 写入内部记忆（session-level context）
- **不输出任何提示**
- 标记该实体为 R1（已标记，待升级）

为什么静默：用户只是顺带提到一个人，不等于在讨论这个人。R1 标记是为 R2 做准备，不是触发分析。

## 阶段 2：二次提及 → R2 提示

当同一实体在同一 session 中被第二次提及时：

- 输出一句简短提示，格式：`（检测到"人名"关联上下文，是否展开？）`
- 如果用户说"展开"/"细说"/"对" → 调出 core-memory 该实体的完整上下文
- 如果用户忽略 → 不追问，继续当前话题

R2 的作用是让用户知道"系统注意到这个人的重复出现了"，而不是替用户做判断。

## 阶段 3：三次及以上 → R3 混沌标记

当同一实体在同一 session 中被第三次及以上提及，且话题围绕该实体展开递归讨论时：

- 按 COGITATE 混沌协议的 R3 刺醒逻辑处理
- 写入 `logs/chaos/CHAOS_YYYY-MM-DD.md`
- 同步 Trigger Log

## 跨模块约束

| 当前激活的技能 | 监听器行为 |
|-------------|----------|
| ELEMENT (Quick) | 监听器静默运行，不输出 R2。用户说"真累人"然后提到"冥后"——这不是展开的时机。等下一轮。 |
| ELEMENT (Warm) | 允许 R2 提示。暖层态下用户正在分享真实连接，监听器可以轻声提醒。 |
| COGITATE | 全功能运行。R1/R2/R3 按协议执行。 |
| REPAIR-SELF | 监听器静默运行。痛苦修复中不插播关联话题，除非用户自己转向。 |
| IDEALISM | 监听器暂停。内核保护区的优先级高于一切上下文关联。 |
| BOURDIEU | 监听器提供关联上下文但不主动触发 R2。社会学分析中的实体引用是材料不是信号。 |

## 生命周期

- 监听目标注册：session 启动时一次
- R1→R2→R3 状态：session 内递增，session 结束后清零
- 不跨 session 持久化监听状态（core-memory 本身是持久化的，但提及计数不是）
