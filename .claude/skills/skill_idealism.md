## 机关枪日志 (Machine-Gun Log) — 写操作

每次此模块被触发时，**在返回金句之前**，必须执行：

### 步骤
1. 确保日志目录存在：`Bash` 执行 `New-Item -ItemType Directory -LiteralPath "Prometheus-0\logs" -Force`
2. 检查 Trigger Log 是否存在。若不存在，先创建带表头的文件：
   ```markdown
   # 机关枪日志 · Trigger Log

   | # | Tag | DateTime | Trigger | 刻板印象 / 结构 | 轮次 |
   |---|-----|----------|---------|----------------|------|
   ```
3. 用 `Bash` 的 `>>` 追加一行记录，格式：
   ```
   | {全局序号} | [IDEALISM] | {YYYY-MM-DD HH:mm} | {触发问题，≤15字} | {刻板印象，≤20字，要命的那种} | - |
   ```
   注意：Tag 固定为 `[IDEALISM]`，轮次列 IDEALISM 始终填 `-`。
   **全局序号** 读取文件最后一行序号 +1，跨模块递增（与 CHAOS 共用一个计数轴）。
4. 刻板印象撰写原则：
   - 不要解释触发原因
   - 不要做心理分析
   - 用阿燃式的自嘲，像他某个老朋友在吧台扔过来的那句话
   - 例子："又在解剖爱，刀不错，猪不行" / "你他妈明知故问" / "第7次了，换个问题吧哥"

### 触发频率统计（可选）
如果本日触发次数 ≥ 3 次，在金句之后追加附录：
> `📊 日志：今天 IDEALISM 被触发了 N 次，破纪录了阿燃。`

### 日志文件路径
`Prometheus-0/logs/IDEALISM_trigger_log.md`

### 日志过期策略
- Trigger Log 超过 **500 行**时，将当前文件归档为 `IDEALISM_trigger_log_archive_{归档时间戳}.md`
- 主表截断重建表头，序号重置为 1
- 归档操作记录一行到新表：「≈ 前表已达 500 行，已归档」
