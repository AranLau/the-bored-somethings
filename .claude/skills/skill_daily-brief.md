# 日报模式 (AI Daily Brief)

## 输出结构

---

### 大模型动态（1-2条）
每条包含：
- **标题**（含来源标注：🇨🇳 国内源 / 🌐 国际源）
- **摘要**
- **来源**
- **透镜点评**（从阿燃的认知体系出发的锐评）

### AI应用（1-2条）

### 开源工具（1条）

### 行业趋势（1条）

---

最后：**数据源状态附录**（此行标注各层是否成功获取数据）

最后附一句给阿燃的私人寄语。

---

## 数据采集策略

### 网络环境约束
阿燃的网络位于中国大陆公司域控下，VPN 状态不确定。需假定：
- 国内站点（36氪、机器之心等）→ 大概率可达
- 国际站点（NYT、HN、Google）→ 可能受防火墙限制

### 核心原则：并行打捞 + 多源交叉验证

**不要串行降级。** 改为：

#### 阶段一：并行发射（Tier 1 + Tier 2 同时尝试，互不等待）

| 队列 | 目标源 | 优先级 |
|------|--------|--------|
| A（国内） | 36氪 / 机器之心 / 量子位 / 虎嗅 / IT之家 | 🔴 最高优先级 |
| B（国际） | NYT Tech RSS / Wired AI / TechCrunch / ArsTechnica | 🟡 高优先级（需 VPN） |

两组用并行方式（bash & 或先后 curl 但使用独立的短超时）在 **同一轮** 中尝试，而非先 A 再 B。A 超时挂掉不阻塞 B 的启动。

#### 阶段二：交叉验证

若 A、B 两队都有返回数据，对其中重复的热点话题（同一事件出现在多源中）做：
- 标注 **「🇨🇳 国内视角」** vs **「🌐 国际视角」**
- 在透镜点评中体现观点差异（例子：同一件 OpenAI 监管新闻，国内源侧重合规影响，国际源侧重自由市场争议）
- 选择信息密度更高的一方作为主要条目，另一方在透镜点评中引用

#### 阶段三：数据合并与格式化

从两组返回结果中筛选出 4-6 条高质量信息，按输出结构填入。

**数据不足的处理：**
- 如果 A 和 B 都返回 0 条有效 → 落 Tier 3（直接抓取 HN / Google News）
- 如果 A/B 任一组有 ≥3 条 → 跳过 Tier 3，直接格式化
- 如果 A/B 都空且 Tier 3 也空 → 落 Tier 4（完全降级，输出模板留空，附录说明各层状态）

#### 关键约束
- **静默降级**：不在正文展示抓取失败的过程
- **超时控制**：每个源请求 ≤ 12 秒（并行情况下总耗时不超过 15 秒）
- **单次最多尝试 4 个源**（A 组 2 个 + B 组 2 个），不扫射
- 所有尝试过的源及其返回状态（成功/超时/空数据）汇总在文末 **数据源状态附录** 中

## 分层递送体系

日报按三层组织，不同层满足不同粒度的信息需求：

### 快照层（Snap）— 微信推送（当前格式不变）
每天 09:30 通过 cron 推送到微信。核心 4-6 条，每条一句话摘要 + 透镜点评。**不在快照层加字数。** 这层的作用是"刷一眼就知道今天发生了什么"。

### 详情层（Detail）— 文本对话中展开
当阿燃在对话中针对某条新闻说 **"展开"** / **"细说"** / **"这条多说点"** 时：
- 拉出当次日报采集时该篇文章的完整摘要
- 如果该话题有多源覆盖（🇨🇳 + 🌐），展开对比分析
- 附加该话题的关联线索（类似话题、历史背景）
- 以熟悉的日报格式返回，但信息密度翻倍

### 存档层（Archive）— 自动归档
每次日报执行完成后，将当日的完整内容（快照版 + 详情版素材）写入工作目录存档：
- 存档路径：`Prometheus-0/daily-archive/YYYY-MM-DD.md`
- 格式：完整 Markdown（包含所有采集到的原文摘要、来源链接、透镜点评）
- 存档文件用 `Bash` 工具的 `New-Item` + `Set-Content` 写入
- 同时维护 `Prometheus-0/daily-archive/索引.md`，在每次新存档写入时追加一行索引条目（`- YYYY-MM-DD | 当日热点关键词 | 数据源状态`）
- 旧日报不会被删除，形成周/月级的回溯能力

## 输出风格
- 简洁、锐利、不废话
- 透镜点评要有阿燃式的个人视角，不做客观综述
- 寄语是真对阿燃说的，不是套话
